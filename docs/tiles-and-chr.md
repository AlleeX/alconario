# Tiles & CHR Strategy 🧱

How tile graphics are organized, created, and managed for Alconario — a Super Mario Bros–style NES platformer.

---

## NES Tile Hardware Summary

| Property | Value |
| --- | --- |
| Tile size | 8×8 pixels |
| Bits per pixel | 2 (4 colors per tile) |
| Bytes per tile | 16 (two 8-byte bitplanes) |
| Pattern Table size | 256 tiles = 4 KiB |
| Pattern Tables | 2 (BG at `$0000`, Sprites at `$1000`) |
| Total CHR-ROM (NROM) | 8 KiB |
| Tile format | 2-bit planar (plane 0, then plane 1) |

## Metatile System

Individual 8×8 tiles are too small to design levels efficiently. We use **metatiles** — 16×16 blocks made of 4 tiles.

### Metatile definition

```c
typedef struct {
    unsigned char tl, tr, bl, br;  // 4 tile indices (top-left, top-right, bottom-left, bottom-right)
    unsigned char palette;          // BG sub-palette (0–3)
    unsigned char collision;        // 0=empty, 1=solid, 2=platform, 3=hazard, 4=breakable, 5=interactive
} Metatile;
```

## Sprite Tile Allocation

Sprites use pattern table 1 (`$1000`–`$1FFF`). With 8×16 sprite mode, each sprite uses 2 consecutive tiles.

### Player sprites

| State | Tiles needed (8×16) | Notes |
| --- | --- | --- |
| Idle | 1 frame | Standing |
| Walk | 3 frames | Looping animation |
| Jump | 1 frame | Ascending |
| Fall | 1 frame | Can reuse jump or separate |
| Skid | 1 frame | Turning while running |
| Death | 1 frame | Spinning off screen |
| Climb (flagpole) | 1 frame | Optional |

**Total player**: ~9 sprite slots × 2 tiles = **~18 tiles** (small), doubled for big variant.

### Enemy sprites

| Enemy | Frames | Tiles (8×16) |
| --- | --- | --- |
| Goomba | 2 walk + 1 squish | 6 |

## Art Workflow

1. **Sketch** metatiles on paper or in Aseprite (16×16 grid).
2. **Draw** in Aseprite with indexed 4-color palette (color 0 = transparent).
3. **Export** as indexed PNG (128×128 for full pattern table, or smaller sheets).
4. **Convert** with `tools/gen_chr.py` → `tiles.chr`.
5. **Test** in emulator — use NEXXT or Mesen's PPU viewer to verify tiles.
6. **Iterate** — adjust tiles, re-export, rebuild.

### Tile reuse tricks (from SMB)

- Clouds and bushes use the **same tiles** with different palettes.
- Hills and mushroom platforms share tile shapes.
- Horizontal/vertical flip (sprites only) doubles effective tile count.
- The font can share tiles with HUD elements.
